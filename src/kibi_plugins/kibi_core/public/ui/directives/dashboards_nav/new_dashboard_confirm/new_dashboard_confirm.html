Dashboards with a Saved Search will:

  • Display a count of results (e.g. the hits on that saved search)
  • Be easier to populate with meaningful Widgets - filtering by clicking on the widget will always work coherently
  • Can be Target or Origin of relational navigation functionalities (Set to Set navigation)

Dashbords without Saved Search will be typically used for cross index "summary pages".

Note: To create a saved searchs from one of your Index Patterns use the Visualize Page.
